# Neovim 现代工作流与生存指南 (2024+ 版)

本文档结合了速查表（Reference）与基于场景的生存指南（Survival Guide），旨在帮助你快速找回“肌肉记忆”，并充分利用现代 Neovim 强大的 Lua 生态插件（如 `fzf-lua`, `Treesitter`, `Grapple`, `Trouble` 等）。

---

## ⚡ 第一部分：快速复健指南（TL;DR）

太久没写代码，忘了所有快捷键？记住这几条最重要的生存法则：

1. **终极救场**：在普通模式下按下 `<space>`（即 `<leader>`）或者 `,`（即 `<localleader>`），停顿一秒钟，屏幕下方会弹出 `Which-key` 菜单，提示你所有可用的快捷键。
2. **极速保存与退出**：
   - `XX` -> 快速保存当前文件。
   - `QQ` -> 关闭所有并退出。
   - `jk` -> 插入模式下极速回到普通模式。
3. **窗口与多文件管理**：
   - `<C-h/j/k/l>` -> 在 Nvim 的多个 Split 之间切换（无缝集成 Tmux）。
   - `<C-n>` / `<C-p>` -> 切换到 下一个 / 上一个 Buffer。
   - **`<C-c>` -> 只关文件不关窗**。关闭当前 Buffer，但保留 Split 窗口布局不被破坏。
4. **找文件与搜代码（核心：`fzf-lua`）**：
   - `<leader>f` -> 在当前工作区找文件。
   - `<leader>s.` -> **极其好用**：仅在“当前正在编辑的文件所在的文件夹”里找其他文件。
   - `<leader>/` -> 全局 Live Grep 代码。
   - `<leader>sw` -> 搜索光标当前所在的单词。
5. **代码诊断排错（`Trouble`）**：
   - `<leader>xx` -> 弹出 Trouble 面板，列出整个工程的报错和警告。
   - `[d` / `]d` -> 不开面板，直接在代码中跳转到上一个/下一个错误行。

## 🚀 第二部分：进阶核心工作流

### 1. 闪电移动与代码跳转 (Leap & Flash)
抛弃狂按 `j` 和 `k` 的旧习惯，使用下面的大杀器：
- `s` -> **Leap 跳转**：按下 `s`，然后输入屏幕上任意位置的前两个字母，按提示的标签字母直接瞬移。
- `S` -> **Flash Treesitter**：通过语法树高亮出当前光标外的函数、块或者 if 语句。输入提示字母直接选中该代码块。
- `r` -> **Remote Flash**（操作符待定模式）：例如你想复制远处的单词，输入 `yr` (yank remote)，选远处单词，光标甚至无需移动过去即可完成复制。

### 2. 热点文件打标 (Grapple)
替代传统的 vim marks，特别适合在多个核心文件之间反复横跳：
- `<leader>ma` -> 将当前文件“加入收藏 / 打上书签”。
- `<leader>mk` 或 `<leader>M` -> 弹出书签列表弹窗。
- `<leader>n` -> 在你收藏的书签文件之间无缝轮切（Cycle）。

### 3. 文本编辑与环绕 (Surround & Unimpaired)
写代码时修改括号、引号、增加空行是最高频的操作。

**环绕字符操作 (`nvim-surround`)**
| 用途 | 快捷键 | 示例操作 | 说明 |
|------|--------|----------|------|
| **添加**环绕 | `ys` + 动作 + 符号 | `ysiw"` | 将 `word` 变成 `"word"` (Yank Surround) |
| **修改**环绕 | `cs` + 旧符号 + 新符号 | `cs"'` | 将 `"word"` 变成 `'word'` (Change Surround) |
| **删除**环绕 | `ds` + 符号 | `ds"` | 将 `"word"` 变成 `word` (Delete Surround) |
| 选中文本添加 | 视觉选中 + `S` + 符号 | `S{` | 将选中的多行文本用 `{}` 括起来 |

**成对快捷键 (`vim-unimpaired`)**
| 用途 | 快捷键 | 说明 |
|------|--------|------|
| 插入空行 | `[<Space>` / `]<Space>` | 在当前行上方/下方插入空行，保持普通模式 |
| 移动当前行 | `[e` / `]e` | 将当前行向上/向下交换位置 |

### Treesitter 结构化对象与移动
基于语法树，文本对象更加智能（操作待定模式和可视模式均有效）：

**核心文本对象：**
- `c`：类 (Class)
- `m`：方法/函数 (Method)
- `f`：函数调用 (Function call)
- `i`：条件语句 (If)
- `l`：循环 (Loop)
- `a`：参数 (Argument)
- `=`：赋值语句 (Assignment)

**实战示例：**
- `cr=`：修改赋值的右侧 (Change RHS of =)
- `caa`：修改一个参数
- `cii`：修改 if 内部（根据光标自动选中条件或条件体）
- `daf`：删除一个函数调用

**结构化跳转：**
所有对象都支持通过 `[` 和 `]` 跳转到上一个/下一个。
- `]m`：下一个函数开头
- `]c`：下一个类开头
- `[f`：上一个函数调用开头

*(注：可用 `;` 和 `,` 重复执行上述的 Treesitter 结构化跳转)*

### 代码折叠 (Folding)
基于 Treesitter 的原生折叠，能够精准识别类、方法、代码块，而不是简单依赖缩进。

**基础开关（最常用）：**
- `za` -> 切换折叠（Toggle）：展开或折叠当前光标所在的代码块。
- `zM` -> 全部折叠（Close All）：将当前文件所有可折叠的代码全缩起来（方便看骨架）。
- `zR` -> 全部展开（Open All）：展开文件里的所有折叠。

**精细控制：**
- `zc` -> 折叠当前块 (Close)
- `zo` -> 展开当前块 (Open)

**跳转折叠：**
- `zj` -> 跳转到下一个折叠块的开头
- `zk` -> 跳转到上一个折叠块的开头
- `]z` -> 使用 Treesitter 语义精确跳转到下一个折叠
- `[z` -> 使用 Treesitter 语义精确跳转到上一个折叠

### 大纲与排错
- **Aerial (大纲)**：`<leader>a` 切换右侧结构树。使用 `<leader>{` 和 `<leader>}` 在结构树（如 YAML 同级节点、类方法）间快速上下跳跃。
- **Trouble (排错)**：`<leader>xx` 看工程报错，`<leader>xX` 看当前文件报错，`<leader>xq` 看 Quickfix 列表。
